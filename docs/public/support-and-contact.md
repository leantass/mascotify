# Soporte y contacto minimo de Mascotify

Este documento deja preparada una base minima y seria de soporte para salida
publica. No reemplaza una operacion formal, pero evita publicar el producto sin
un canal claro para consultas.

## 1. Canal minimo recomendado

Antes de publicar, definir al menos uno de estos canales:

- correo de soporte
- formulario publico de contacto
- pagina publica de soporte

Lo ideal es que el mismo canal figure en:

- web publica
- ficha de Play Store
- ficha de App Store, si aplica

## 2. Que deberia cubrir ese canal

Como minimo, el canal de soporte deberia aceptar:

- consultas generales del producto
- problemas de acceso o sesion
- consultas sobre datos o privacidad
- reportes basicos de funcionamiento

## 3. Tiempo de respuesta

No conviene prometer SLA ni tiempos estrictos si todavia no existe una
operacion formal. La salida minima puede indicar una respuesta razonable segun
disponibilidad del canal publicado.

## 4. Datos a completar antes de publicar

- Nombre de contacto o responsable: completar
- Correo de soporte: completar
- Correo de privacidad: completar
- URL publica de soporte, si existe: completar

## 5. Texto base sugerido para web o stores

"Si necesitas ayuda con Mascotify o quieres realizar una consulta sobre la app,
usa el canal de soporte oficial publicado junto con esta experiencia."

## 6. Pendiente fuera del repo

Este documento no crea por si solo un canal operativo. Antes de publicar hay
que completar y activar el medio de contacto real que se va a mostrar al
usuario final.
